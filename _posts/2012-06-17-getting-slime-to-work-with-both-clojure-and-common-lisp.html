--- layout: post title: Getting Slime to Work With Both Clojure and Common Lisp published: true ---

Though I don’t have any large projects in Common Lisp I’ve been reading an book on AI which is filled with programs written in the language. Having had the pleasure of using slime and swank when programming in Clojure, I knew that I wanted to use them again when working through the code in the book. As soon as I tried to do so I realized that I had a problem on my hands.

As it turns out, the slime used by Clojure and the slime used by Common Lisp aren’t exactly the same. The Clojure slime crashes when trying to connect to a swank server for common lisp and vice versa. Luckily I ended up finding Arnab Deka’s site. Arnab encountered the same problem as me and had already solved the issue.

So the only problem I had was that clisp was that my clisp wasn’t in usr/local/bin/clisp like it was in the github link he posted. I had installed my clisp with sudo apt-get install clisp and it hadn’t put it there. I found out where it had been put by running find | grep bin/clisp from the / directory. It ended up being in usr/bin/clisp. Once I had changed the bit of lisp code he supplied to reflect this M-x common-lisp-slime started working like it was supposed too.
